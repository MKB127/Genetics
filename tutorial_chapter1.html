Chapter 1

Understanding Pedigrees

A pedigree is a standardised diagram representing family relationships across multiple generations. This chapter covers the core concepts you need before working with the app.

What is a Pedigree?

The ped Object
Internally, a pedigree is encoded as 4 vectors of equal length — one entry per individual.
id
Individual labels — unique identifiers for every person in the pedigree.
fid
Father ID — ID of the father, or 0 if unknown (founder).
mid
Mother ID — ID of the mother, or 0 if unknown (founder).
sex
Sex code — 1 = male, 2 = female, 0 = unknown.

Pedigree Title
Every pedigree has a title displayed above the diagram — the family name or case identifier. It is set when you create or load a pedigree.
✏️

Custom IDs: While the app defaults to standard Roman/Arabic notation, you can assign any custom label to each individual. The app preserves your labels throughout the analysis.

International Standard

PNPG / Bennett 2008 guidelines define symbols, line types, and notation. Squares = males, circles = females, diamonds = unknown, horizontal lines = couples, vertical lines = children.

What the App Implements

The app follows these conventions as closely as technical limits allow: correct shapes, generation-indexed IDs, couple lines, descent lines, and all standard status markers.

Custom Extensions

The app extends the standard with custom phenotypes (fill colours, hatch patterns, motif symbols) and free-form individual labels for clinical or research use.

Custom Phenotypes

Beyond standard symbols, the app lets you create custom phenotypes to visually tag any subset of individuals — affected status, carrier state, or any trait of interest. Two types are available:

🎨 Fill

Colours the background of the individual's symbol. Can include a hatched pattern to indicate partial or carrier status.

🔣 Motif

Overlays a small symbol (half-circle, dot, cross…) at a chosen position. Useful when tracking multiple phenotypes simultaneously.
